## Design Context

### Users
UX researchers running affinity mapping workshops with stakeholders. They need a collaborative canvas for synthesizing user interview insights, clustering observations, and running dot voting sessions to identify key themes. The app is used in workshop settings (live collaboration) and solo synthesis work.

### Brand Personality
**Playful & Creative** - The interface should feel approachable and energizing, not sterile. Think post-it notes come to life - there's joy in organizing ideas. The green/orange palette already signals this. Animation should feel springy and natural, not rigid.

### Aesthetic Direction
**Figma/Miro style** - Clean canvas with subtle shadows, rounded corners, and purposeful whitespace. The interface should get out of the way so users can focus on their content. Color picker with semantic labels (Insight, Pain Point, Quote) adds personality without being distracting.

**Anti-references**: Avoid generic AI aesthetic - no cookie-cutter gradient backgrounds, no "AI slop" patterns. The design should feel hand-crafted and intentional.

### Theme Support
Light and dark mode supported via CSS variables. Dark mode uses brighter greens to maintain the playful feel while ensuring readability.

### Design Principles

1. **Canvas-first** - The workspace is the product. Toolbar and panels should feel lightweight, not overwhelming. Use slide-over panels rather than modals where possible.

2. **Semantic colors over arbitrary ones** - Sticky note colors have meaning (Insight, Pain Point, Quote). Help users organize by giving colors purpose.

3. **Springy, natural motion** - Animations use spring physics, not linear easing. Things should feel tactile and responsive to user input.

4. **Collaborative joy** - Real-time collaboration should feel magical. Activity indicators, presence, and sync should make working together feel effortless.

5. **Respect the content** - Stickies, clusters, and comments are user work. Interface chrome should never compete with user content for attention.
